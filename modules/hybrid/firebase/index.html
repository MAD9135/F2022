# Firebase

Firebase is a collection of services run by Google. The services include:

  1. Firestore - a document database similar to Mongo that you can directly communicate with from the browser with JavaScript.
  2. Hosting - actual hosting of websites
  3. Authentication - the ability to add username and password authentication to your website as well as support for OAuth with Github, Facebook, Twitter, and others.
  4. Storage - storage of files like audio, video, images, etc.
  5. Cloud Functions - serverless style functions that can talk to APIs remotely.
  6. Cloud Messaging - Push messaging for your PWAs.

and others.

There are other service collections that are similar. Amazon has AWS which has related services for storage, hosting, cloud functions etc. Microsoft has Azure. Google also has Google Cloud Platform. IBM has their IBM Cloud Services that includes Watson.

Shortly before this program started if you wanted to host your website, you needed to have familiarity with Dev Ops, setting up a database, using FTP to transfer files to server storage locations, plus multiple programming languages for the server-side scripts that controlled your website.

Today, server-side is more commonly approached as a Service. You pay for the processing time you need on your provider's computers plus the amount of network traffic you use. The Dev Ops part is taken care of by the company that owns the servers. This is the Service that you are paying for.

# Getting Started with a Firebase Project

To be able to use the Firebase services you need to sign up for a Firebase account.

First, visit https://firebase.google.com/ (opens new window) and sign up for a Google Account, or sign in with an existing Google Account.

Next, we will want to go to the Firebase Console (opens new window). This is where you will find your projects and apps.

# Local SetUp & Webpack

We will be using version 9 of Firebase, which changed the approach to writing our code to take advantage of ES Modules. It is a more modular version than version 8, which means we only import the parts that we need.

It also means that we need to use a module bundler, like WebPack, just like we do with React. To use WebPack we need to use npm and create a project. Here are the basic steps to get started with WebPack for a non-react-based Firebase project.

Create a folder for your sample project. Open the folder in VSCode. Then open the Terminal, which should be pointing to your opened folder. Type the following command to make your folder into an npm-based project and create the package.json file.

npm init - y
1

Next, we want to install the webpack development dependencies.

npm install webpack webpack-cli -D
1

Webpack will want you to have two root folders - src and dist. Just like a React project, we will write all our code in a src folder and the dist folder will be the compiled version that we would run in the browser.

Webpack needs to have its own configuration file called webpack.config.js.

const path = require('path'); //required nodeJS module

module.exports = {
  mode: 'development',
  entry: './src/index.js', //the starter file for our script
  output: {
    path: path.resolve(__dirname, 'dist'), //output folder for the project
    filename: 'bundle.js',
    //name of the compiled JS file inside dist - to be loaded by index.html
  },
  watch: true, //hot reloading when changes are saved inside src folder
};

Now we can create the index.js file inside of src/ and a basic index.html file inside of dist/. The index.html file should include a script tag that points to ./bundle.js. This way, our webpage will load the compiled version of our script modules.

Finally, we want to add a script in package.json so that we can run webpack and have it compile all our script into the build folder.

In your package.json file, update the "scripts" section to be this:

"scripts":{
  "build": "webpack"
}

After this is added you can use the CLI command:

npm run build
1

and it will keep webpack running in the background to recompile your project every time you make a change to index.js.

After webpack has run you can use Live Server in VSCode to open /dist/index.html in the browser to see your project running.

Use Ctrl+C to stop webpack running.

These steps are the most minimal startup steps for any webpack driven project. We still don't have any Firebase code but that comes next. Head over to the FireStore section to learn what to do next.
